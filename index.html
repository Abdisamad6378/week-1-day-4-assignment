Frequently Asked Questions

What is semantic HTML?

Semantic HTML elements are tags that clearly describe their meaning to both the browser and the developer (e.g., <header>, <nav>, <article>, <section>, and <footer>). They make web pages more accessible, readable, and structured for search engine optimization (SEO).

What is the difference between section and aside?

A <section> represents a thematic grouping of content, typically with a heading, and represents main document areas. An <aside> represents content that is tangentially related to the main content around it (like sidebars, quote callouts, or advertising blocks).

Do I need JavaScript for details and summary tags?

No! The <details> and <summary> tags provide native toggle interactions. When clicked, the browser toggles the 'open' attribute on the details tag, expanding or collapsing the contents automatically.

When should I start learning JavaScript?

It's best to start learning JavaScript after you're comfortable structuring layouts with HTML and styling them with CSS. I am starting JavaScript this weekend to begin building dynamic, fully interactive elements!
